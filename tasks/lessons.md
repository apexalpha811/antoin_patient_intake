## Lessons

- When an external autocomplete API is unreliable, keep the form fields as normal editable inputs first. Load autocomplete as optional enhancement, and route load, auth, and constructor failures back to manual input.
- When the app is opened via `file://`, do not load Google Places at all. Local file preview should stay manual-only, while hosted `https://` deployments can still enhance the same fields with autocomplete.
- When PDF export needs stable layout, build a hidden export clone and render that instead of the live screen. That keeps loading text, buttons, and transient UI state out of the file and makes page width predictable.
- When Playwright cannot launch its bundled browser in this workspace, attach to a headless Chrome started with remote debugging instead of stalling on the launch error.
- When a browser-side DOCX export starts producing wrapper-like or pseudo-document output, remove the DOCX path instead of trying to salvage it. Keep the stable PDF and submit flows, then add DOCX back only if a real Word output path is proven end to end.
- When adding office notifications to a form submit, keep the PDF download first, then send the notification, and branch the banner copy by success or failure instead of inventing a second submit flow.
- When PDF output shows blank, dashed, or inconsistent text inside inputs, stop relying on live form controls in the export clone. Replace text inputs and textareas with static text nodes before capture so html2canvas renders the value, not the browser widget chrome.
- When PDF pages clip at the bottom even though page breaks exist, check whether the paginator is treating a whole layout wrapper as one block. Flatten wrapper grids and paginate smaller semantic blocks before reaching for bigger page-height changes.
- When patient-entered textarea content can grow during PDF export, split oversized static textarea blocks by measured word chunks across new PDF pages. Do not slice a rendered canvas after capture, because that can create blank trailing pages and cut text mid-line.
- When fitting textarea text inside a PDF export clone, wait until the block has its final page width before measuring. Detached or pre-layout clones can report the wrong height and leave the font at its base size.
- When PDF export CSS uses `!important` on field typography, shrink-to-fit code must also write `font-size` with `!important`. Plain inline font sizing can lose the cascade and make the fit logic look broken.
- When the Node REPL cannot write into the workspace but PowerShell can, stage binary exports in `C:\\tmp` and copy them back with `Copy-Item` instead of fighting the bridge permissions.
- When capturing a jsPDF export as a data URI, do not assume the string starts with `data:application/pdf;base64,`. Some builds include a `filename=...` prefix before the base64 payload, so split on the first comma.
- When a consent page needs to stay compact, do not force a fresh page too early. A clean-looking break can push the tail blocks onto an extra sheet, so verify the full render before locking it in.
- When a checkbox section has too much dead air, shrink the PDF-only `min-height` on the checkbox labels before touching the surrounding page breaks. That usually gives back more page space with less collateral damage.
- When an image still splits across PDF pages after cropping or resizing the asset, the real fix is usually the containing block or page break, not the image file. Treat the asset change as visual cleanup only.
- When a file-based HTML page uses `html2canvas` and a local image taints the export canvas, inline that image as a data URI in the export clone. Fetching the file URL from `file://` will not save you.
- When a user says to remove text from a section, confirm whether they mean the whole section or just the section heading before deleting markup. A quick assumption can erase the wrong page.
- When one field should stay blank while similar fields still autofill, remove the shared sync hook entirely for that field instead of trying to override it after the fact. Shared update functions will keep winning.
- Autofill pattern: Use `data-SOURCE-source` on the source field and `data-SOURCE-target` on target field(s), then create a sync function that reads the source and writes to all targets. Hook the sync into the source field's input/change events, and include the sync in `refreshAutofilledDisplays()` for export and submit flows.
- When PDF input field text appears bottom-weighted in the export, CSS-only padding/line-height tweaks may not help. The issue can be in html2canvas rasterization, not the DOM styling. Prefer reducing the change surface and testing with rendered page images after each hypothesis.
- When investigating PDF rasterization issues, render captured pages to PNG with a stable toolchain and inspect them directly before assuming the fix is in the application code. Baseline/raster artifacts often survive multiple DOM-layer fixes.
- Cloning form controls into static nodes preserves values but does not guarantee visual parity if the capture layer changes text metrics. Structural DOM changes should be validated end-to-end with an actual PDF render, not just by inspecting the clone tree in DevTools.
- When Playwright PDF tests with zero margins pass but user browser prints differ, check whether browser default margins (~0.75") shrink usable width below Tailwind's `md:` breakpoint (768px). The grids collapse to single column even though the CSS works at full width.
- When using `window.print()` with Tailwind responsive grids in print CSS, always add explicit `grid-template-columns` overrides with `!important` for every responsive grid class. The print media viewport doesn't respect Tailwind's breakpoint logic the same way the screen viewport does.
- When iterating on print layout with Playwright, always test with realistic margins (0.75") to catch viewport-width-dependent issues that zero-margin tests mask. Zero-margin tests gave 7 pages but real browser margins push to 9 pages.
- Font sizes should be preserved when shrinking a form for print. Shape elements (diagrams, signature canvases, headers) give back far more vertical space per unit of CSS change than font-size reductions, and they don't hurt readability.
- A single line in a generic `@media print` block (e.g., `.page { page-break-after: always }`) can add multiple wasted pages by forcing breaks between every wizard section. Always scope forced breaks to specific `.new-page` elements, not broad selectors.
- When a "dirty flag" guard is used to prevent autofill from overwriting user edits, only set the flag from the target's OWN user-input handlers. Never copy the source's dirty flag onto the target inside the clone/sync function — that freezes the autofill after the first character because the source's first input already set source.dirty = true, which propagates and blocks all subsequent syncs.
- When sync logic works for discrete events (draw stroke, `pointerup`) but breaks for streaming events (typed characters, `input`), suspect a state flag that gets latched on the first event and never reset. The discrete path completes in one call before the latch can do damage; the streaming path hits the latch on the second call.
